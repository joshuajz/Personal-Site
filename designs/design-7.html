March 18, 2025 7 min read by Josh Cowan

This post started as a weekend experiment and turned into one of the most fun side projects I've worked on. Building a Discord bot forces you to think about real-time communication, state management, and user experience in ways that typical web apps don't.

I chose TypeScript from the start because the Discord.js v14 types are excellent — they catch mistakes before you ever deploy. Setting up the slash command registration is a bit verbose, but once you have a base class pattern, each new command takes maybe ten minutes to scaffold.

"The real learning came not from the bot itself, but from thinking through the failure modes — what happens when a user spams a button, or a guild goes offline mid-transaction."

The economy system was the most interesting part. I used SQLite for persistence with a simple abstraction layer so I could swap to Postgres later without touching the command logic. Role rewards fire after every transaction threshold is crossed — the users loved it.

If you're thinking about building something similar: start with one command, get the deploy pipeline working, and only then add complexity. The temptation to architect everything up front will kill your momentum.
